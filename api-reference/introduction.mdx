## Overview

MonitorFlow provides a powerful REST API that enables you to programmatically track and monitor events in your application. Our API follows standard REST conventions and uses JSON for request and response payloads.

## Base URL

```bash
https://api.monitorflow.com/v1
```

## Authentication

All API requests require authentication using an API key. You can obtain your API key from the [MonitorFlow Dashboard](https://app.monitorflow.com/settings/api).

Include your API key in the `Authorization` header of your requests:

```bash
Authorization: Bearer your_api_key_here
```

## Request Format

API requests should be made with the following headers:

```bash
Content-Type: application/json
Authorization: Bearer your_api_key_here
```

## Example Request

```javascript
// Track a new event
const response = await fetch('https://api.monitorflow.com/v1/events', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'user.signup',
    priority: 'high',
    metadata: {
      userId: '123',
      email: 'user@example.com'
    }
  })
});

const data = await response.json();
```

## Rate Limiting

API requests are limited to:
- 1000 requests per minute for paid plans
- 100 requests per minute for free plans

Rate limit information is included in the response headers:
```bash
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```
