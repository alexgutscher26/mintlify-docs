## How It Works

### 1. Content Detection

<Steps>
  <Step title="Page Analysis">
    The extension automatically detects when you're on a supported social media platform
  </Step>
  <Step title="Reply Context">
    When you click reply on a post, the extension identifies the original content
  </Step>
  <Step title="UI Injection">
    ReplyIQ interface is seamlessly injected into the platform's reply interface
  </Step>
  <Step title="Context Extraction">
    Original post content, author information, and conversation context are extracted
  </Step>
</Steps>

### 2. Reply Generation Process

<Steps>
  <Step title="User Input">
    You select your preferred tone and any custom instructions
  </Step>
  <Step title="API Request">
    Extension sends the context and preferences to ReplyIQ servers
  </Step>
  <Step title="AI Processing">
    Advanced AI models generate contextually appropriate replies
  </Step>
  <Step title="Response Display">
    Generated replies are displayed with options to edit, regenerate, or use
  </Step>
</Steps>

### 3. Integration & Posting

<Steps>
  <Step title="Review & Edit">
    Review the generated reply and make any necessary edits
  </Step>
  <Step title="Emoji Enhancement">
    Add suggested emojis or hashtags to improve engagement
  </Step>
  <Step title="One-Click Insert">
    Click "Use Reply" to insert the content into the platform's text field
  </Step>
  <Step title="Post Normally">
    Use the platform's native posting functionality to publish your reply
  </Step>
</Steps>

## Privacy & Security

### Data Handling

<AccordionGroup>
  <Accordion title="What Data is Processed">
    - Original post content (for context)
    - Generated reply content
    - User preferences and settings
    - Usage analytics (anonymized)
    - Error logs for debugging
  </Accordion>

  <Accordion title="What Data is NOT Collected">
    - Personal browsing history
    - Private messages or DMs
    - Passwords or login credentials
    - Financial information
    - Unrelated website content
  </Accordion>

  <Accordion title="Data Security">
    - All data encrypted in transit (HTTPS/TLS)
    - No sensitive data stored locally
    - Secure API communication
    - Regular security audits
    - GDPR compliance
  </Accordion>
</AccordionGroup>

### Permissions Explained

<AccordionGroup>
  <Accordion title="Host Permissions">
    **Why needed:** Access social media websites to inject ReplyIQ interface
    
    **Specific domains:**
    - twitter.com, x.com
    - linkedin.com
    - facebook.com
    - instagram.com
    - app.replyiq.com (for API communication)
  </Accordion>

  <Accordion title="Storage Permission">
    **Why needed:** Save user preferences and cache data locally
    
    **What's stored:**
    - User preferences (tone, model selection)
    - Authentication tokens
    - Cached emoji data
    - Usage statistics
  </Accordion>

  <Accordion title="Active Tab Permission">
    **Why needed:** Access current tab content for context extraction
    
    **Usage:**
    - Extract original post content
    - Inject ReplyIQ interface
    - Monitor page changes
    - Detect platform type
  </Accordion>
</AccordionGroup>
